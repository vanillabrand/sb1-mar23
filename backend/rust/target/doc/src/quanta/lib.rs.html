1//! Performant cross-platform timing with goodies.
2//!
3//! `quanta` provides a simple and fast API for measuring the current time and the duration between
4//! events.  It does this by providing a thin layer on top of native OS timing functions, or, if
5//! available, using the Time Stamp Counter feature found on modern CPUs.
6//!
7//! # Design
8//! Internally, `quanta` maintains the concept of two potential clock sources: a reference clock and
9//! a source clock.
10//!
11//! The reference clock is provided by the OS, and always available.  It is equivalent to what is
12//! provided by the standard library in terms of the underlying system calls being made.  As it
13//! uses the native timing facilities provided by the operating system, we ultimately depend on the
14//! OS itself to give us a stable and correct value.
15//!
16//! The source clock is a potential clock source based on the Time Stamp Counter feature found on
17//! modern CPUs.  If the TSC feature is not present or is not reliable enough, `quanta` will
18//! transparently utilize the reference clock instead.
19//!
20//! Depending on the underlying processor(s) in the system, `quanta` will figure out the most
21//! accurate/efficient way to calibrate the source clock to the reference clock in order to provide
22//! measurements scaled to wall clock time.
23//!
24//! Details on TSC support, and calibration, are detailed below.
25//!
26//! # Features
27//! Beyond simply taking measurements of the current time, `quanta` provides features for more easily
28//! working with clocks, as well as being able to enhance performance further:
29//! - `Clock` can be mocked for testing
30//! - globally accessible "recent" time with amortized overhead
31//!
32//! For any code that uses a `Clock`, a mocked version can be substituted.  This allows for
33//! application authors to control the time in tests, which allows simulating not only the normal
34//! passage of time but provides the ability to warp time forwards and backwards in order to test
35//! corner cases in logic, etc.  Creating a mocked clock can be acheived with [`Clock::mock`], and
36//! [`Mock`] contains more details on mock usage.
37//!
38//! `quanta` also provides a "recent" time feature, which allows a slightly-delayed version of time
39//! to be provided to callers, trading accuracy for speed of access.  An upkeep thread is spawned,
40//! which is responsible for taking measurements and updating the global recent time. Callers then
41//! can access the cached value by calling `Clock::recent`.  This interface can be 4-10x faster
42//! than directly calling `Clock::now`, even when TSC support is available.  As the upkeep thread
43//! is the only code updating the recent time, the accuracy of the value given to callers is
44//! limited by how often the upkeep thread updates the time, thus the trade off between accuracy
45//! and speed of access.
46//!
47//! # Feature Flags
48//! `quanta` comes with feature flags that enable convenient conversions to time types in
49//! other popular crates, such as:
50//! - `prost` - provides an implementation into [`Timestamp`][prost_types_timestamp] from
51//! `prost_types`
52//!
53//! # Platform Support
54//! At a high level, `quanta` carries support for most major operating systems out of the box:
55//! - Windows ([QueryPerformanceCounter])
56//! - macOS/OS X/iOS ([mach_continuous_time])
57//! - Linux/*BSD/Solaris ([clock_gettime])
58//!
59//! These platforms are supported in the "reference" clock sense, and support for using the Time
60//! Stamp Counter as a clocksource is more subtle, and explained below.
61//!
62//! # Time Stamp Counter support
63//! Accessing the TSC requires being on the x86_64 architecture, with access to SSE2. Additionally,
64//! the processor must support either constant or nonstop/invariant TSC.  This ensures that the TSC
65//! ticks at a constant rate which can be easily scaled.
66//!
67//! A caveat is that "constant" TSC doesn't account for all possible power states (levels of power
68//! down or sleep that a CPU can enter to save power under light load, etc) and so a constant TSC
69//! can lead to drift in measurements over time, after they've been scaled to reference time.
70//!
71//! This is a limitation of the TSC mode, as well as the nature of `quanta` not being able to know,
72//! as the OS would, when a power state transition has happened, and thus compensate with a
73//! recalibration. Nonstop/invariant TSC does not have this limitation and is stable over long
74//! periods of time.
75//!
76//! Roughly speaking, the following list contains the beginning model/generation of processors
77//! where you should be able to expect having invariant TSC support:
78//! - Intel Nehalem and newer for server-grade
79//! - Intel Skylake and newer for desktop-grade
80//! - VIA Centaur Nano and newer (circumstantial evidence here)
81//! - AMD Phenom and newer
82//!
83//! Ultimately, `quanta` will query CPUID information to determine if the processor has the
84//! required features to use the TSC.
85//!
86//! # Calibration
87//! As the TSC doesn't necessarily tick at reference scale -- i.e. one tick isn't always one
88//! nanosecond -- we have to apply a scaling factor when converting from source to reference time
89//! scale.  We acquire this scaling factor by querying the processor or calibrating our source
90//! clock to the reference clock.
91//!
92//! In some cases, on newer processors, the frequency of the TSC can be queried directly, providing
93//! a fixed scaling factor with no further calibration necessary.  In other cases, `quanta` will
94//! have to run its own calibration before the clock is ready to be used: repeatedly taking
95//! measurements from both the reference and source clocks until a stable scaling factor has been
96//! established.
97//!
98//! This calibration is stored globally and reused.  However, the first `Clock` that is created in
99//! an application will block for a small period of time as it runs this calibration loop.  The
100//! time spent in the calibration loop is limited to 200ms overall.  In practice, `quanta` will
101//! reach a stable calibration quickly (usually 10-20ms, if not less) and so this deadline is
102//! unlikely to be reached.
103//!
104//! # Caveats
105//! Utilizing the TSC can be a tricky affair, and so here is a list of caveats that may or may not
106//! apply, and is in no way exhaustive:
107//! - CPU hotplug behavior is undefined
108//! - raw values may time warp
109//! - measurements from the TSC may drift past or behind the comparable reference clock
110//!
111//! # WASM support
112//!
113//! This library can be built for WASM targets, but in this case the resolution
114//! and accuracy of measurements can be limited by the WASM environment. In
115//! particular, when running on the `wasm32-unknown-unknown` target in browsers,
116//! `quanta` will use [windows.performance.now] as a clock. This mean the
117//! accuracy is limited to milliseconds instead of the usual nanoseconds on
118//! other targets. When running within a WASI environment (target
119//! `wasm32-wasi`), the accuracy of the clock depends on the VM implementation.
120//!
121//! [QueryPerformanceCounter]: https://msdn.microsoft.com/en-us/library/ms644904(v=VS.85).aspx
122//! [mach_continuous_time]: https://developer.apple.com/documentation/kernel/1646199-mach_continuous_time
123//! [clock_gettime]: https://linux.die.net/man/3/clock_gettime
124//! [prost_types_timestamp]: https://docs.rs/prost-types/0.7.0/prost_types/struct.Timestamp.html
125//! [windows.performance.now]: https://developer.mozilla.org/en-US/docs/Web/API/Performance/now

157// Run 500 rounds of calibration before we start actually seeing what the numbers look like.
158const MINIMUM_CAL_ROUNDS: u64 = 500;
159
160// We want our maximum error to be 10 nanoseconds.
161const MAXIMUM_CAL_ERROR_NS: u64 = 10;
162
163// Don't run the calibration loop for longer than 200ms of wall time.
164const MAXIMUM_CAL_TIME: Duration = Duration::from_millis(200);
165

168enum ClockType {
169    Monotonic(Monotonic),
170    Counter(AtomicCell<u64>, Monotonic, Counter, Calibration),
171    Mock(Arc<Mock>),
172}

174#[derive(Debug, Copy, Clone)]
175pub(crate) struct Calibration {
176    ref_time: u64,
177    src_time: u64,
178    scale_factor: u64,
179    scale_shift: u32,
180}
181
182impl Calibration {
183    pub fn new() -> Calibration {
184        Calibration {
185            ref_time: 0,
186            src_time: 0,
187            scale_factor: 1,
188            scale_shift: 1,
189        }
190    }
191
192    fn scale_src_to_ref(&self, src_raw: u64) -> u64 {
193        let delta = src_raw.saturating_sub(self.src_time);
194        let scaled = mul_div_po2_u64(delta, self.scale_factor, self.scale_shift);
195        scaled + self.ref_time
196    }
197
198    fn calibrate(&mut self, reference: &Monotonic, source: &Counter) {
199        // future improvement: read the TSC frequency directly with something like the
200        // code in this PR: https://github.com/hermitcore/uhyve/pull/24
201
202        let mut variance = Variance::default();
203        let deadline = reference.now() + MAXIMUM_CAL_TIME.as_nanos() as u64;
204
205        self.ref_time = reference.now();
206        self.src_time = source.now();
207
208        let loop_delta = 1000;
209        loop {
210            // Busy loop to burn some time.
211            let mut last = reference.now();
212            let target = last + loop_delta;
213            while last < target {
214                last = reference.now();
215            }
216
217            // We put an upper bound on how long we run calibration before to provide a predictable
218            // overhead to the calibration process.  In practice, even if we hit the calibration
219            // deadline, we should still have run a sufficient number of rounds to get an accurate
220            // calibration.
221            if last >= deadline {
222                break;
223            }
224
225            // Adjust our calibration before we take our measurement.
226            self.adjust_cal_ratio(reference, source);
227
228            let r_time = reference.now();
229            let s_raw = source.now();
230            let s_time = self.scale_src_to_ref(s_raw);
231            variance.add(s_time as f64 - r_time as f64);
232
233            // If we've collected enough samples, check what the mean and mean error are.  If we're
234            // already within the target bounds, we can break out of the calibration loop early.
235            if variance.has_significant_result() {
236                let mean = variance.mean().abs();
237                let mean_error = variance.mean_error().abs();
238                let mwe = variance.mean_with_error();
239                let samples = variance.samples();
240
241                if samples > MINIMUM_CAL_ROUNDS
242                    && mwe < MAXIMUM_CAL_ERROR_NS
243                    && mean_error / mean <= 1.0
244                {
245                    break;
246                }
247            }
248        }
249    }
250
251    fn adjust_cal_ratio(&mut self, reference: &Monotonic, source: &Counter) {
252        // Overall algorithm: measure the delta between our ref/src_time values and "now" versions
253        // of them, calculate the ratio between the deltas, and then find a numerator and
254        // denominator to express that ratio such that the denominator is always a power of two.
255        //
256        // In practice, this means we take the "source" delta, and find the next biggest number that
257        // is a power of two.  We then figure out the ratio that describes the difference between
258        // _those_ two values, and multiple the "reference" delta by that much, which becomes our
259        // numerator while the power-of-two "source" delta becomes our denominator.
260        //
261        // Then, conversion from a raw value simply becomes a multiply and a bit shift instead of a
262        // multiply and full-blown divide.
263        let ref_end = reference.now();
264        let src_end = source.end();
265
266        let ref_d = ref_end.wrapping_sub(self.ref_time);
267        let src_d = src_end.wrapping_sub(self.src_time);
268
269        // TODO: we should almost never get a zero here because that would mean denom was greater
270        // than 2^63 which is already a red flag.. but i'm not 100% sure if we can prove it well
271        // enough to simply keep the panic around? gotta think on this
272        let src_d_po2 = src_d.next_power_of_two();
273        if src_d_po2 == 0 {
274            panic!("po2_denom was zero!");
275        }
276
277        // TODO: lossy conversion back and forth just to get an approximate value, can we do better
278        // with integer math? not sure
279        let po2_ratio = src_d_po2 as f64 / src_d as f64;
280        self.scale_factor = (ref_d as f64 * po2_ratio) as u64;
281        self.scale_shift = src_d_po2.trailing_zeros();
282    }
283}

291/// Unified clock for taking measurements.
292#[derive(Debug, Clone)]
293pub struct Clock {
294    inner: ClockType,
295}
296
297impl Clock {
298    /// Creates a new clock with the optimal reference and source clocks.
299    ///
300    /// Support for TSC, etc, are checked at the time of creation, not compile-time.
301    pub fn new() -> Clock {
302        let reference = Monotonic::new();
303        let inner = if has_tsc_support() {
304            let source = Counter::new();
305            let calibration = GLOBAL_CALIBRATION.get_or_init(|| {
306                let mut calibration = Calibration::new();
307                calibration.calibrate(&reference, &source);
308                calibration
309            });
310            ClockType::Counter(AtomicCell::new(0), reference, source, *calibration)
311        } else {
312            ClockType::Monotonic(reference)
313        };
314
315        Clock { inner }
316    }
317
318    /// Creates a new clock that is mocked for controlling the underlying time.
319    ///
320    /// Returns a [`Clock`] instance and a handle to the underlying [`Mock`] source so that the
321    /// caller can control the passage of time.
322    pub fn mock() -> (Clock, Arc<Mock>) {
323        let mock = Arc::new(Mock::new());
324        let clock = Clock {
325            inner: ClockType::Mock(mock.clone()),
326        };
327
328        (clock, mock)
329    }
330
331    /// Gets the current time, scaled to reference time.
332    ///
333    /// This method is the spiritual equivalent of [`std::time::Instant::now`].  It is guaranteed
334    /// to return a monotonically increasing value between calls to the same `Clock` instance.
335    ///
336    /// Returns an [`Instant`].
337    pub fn now(&self) -> Instant {
338        match &self.inner {
339            ClockType::Monotonic(monotonic) => Instant(monotonic.now()),
340            ClockType::Counter(last, _, counter, _) => {
341                let now = counter.now();
342                // Update the last timestamp with `now`, if `now` is greater
343                // than the current value.
344                // TODO: replace with `AtomicCell::fetch_max` once `crossbeam-utils` implements it.
345                let last = last.fetch_update(|current| Some(current.max(now))).unwrap();
346                // `fetch_max` always returns the previous value, so we need to
347                // see which is *actually* the max.
348                let actual = std::cmp::max(now, last);
349                self.scaled(actual)
350            }
351            ClockType::Mock(mock) => Instant(mock.value()),
352        }
353    }
354
355    /// Gets the underlying time from the fastest available clock source.
356    ///
357    /// As the clock source may or may not be the TSC, value is not guaranteed to be in nanoseconds
358    /// or to be monotonic.  Value can be scaled to reference time by calling either [`scaled`]
359    /// or [`delta`].
360    ///
361    /// If you need maximum accuracy in your measurements, consider using [`start`] and [`end`].
362    ///
363    /// [`scaled`]: Clock::scaled
364    /// [`delta`]: Clock::delta
365    /// [`start`]: Clock::start
366    /// [`end`]: Clock::end
367    pub fn raw(&self) -> u64 {
368        match &self.inner {
369            ClockType::Monotonic(monotonic) => monotonic.now(),
370            ClockType::Counter(_, _, counter, _) => counter.now(),
371            ClockType::Mock(mock) => mock.value(),
372        }
373    }
374
375    /// Gets the underlying time from the source clock, specific to starting an operation.
376    ///
377    /// Value is not guaranteed to be in nanoseconds.
378    ///
379    /// Provides the same functionality as [`raw`], but tries to ensure that no extra CPU
380    /// instructions end up executing after the measurement is taken.  Since normal processors are
381    /// typically out-of-order, other operations that logically come before a call to this method
382    /// could be reordered to come after the measurement, thereby skewing the overall time
383    /// measured.
384    ///
385    /// [`raw`]: Clock::raw
386    pub fn start(&self) -> u64 {
387        match &self.inner {
388            ClockType::Monotonic(monotonic) => monotonic.now(),
389            ClockType::Counter(_, _, counter, _) => counter.start(),
390            ClockType::Mock(mock) => mock.value(),
391        }
392    }
393
394    /// Gets the underlying time from the source clock, specific to ending an operation.
395    ///
396    /// Value is not guaranteed to be in nanoseconds.
397    ///
398    /// Provides the same functionality as [`raw`], but tries to ensure that no extra CPU
399    /// instructions end up executing before the measurement is taken.  Since normal processors are
400    /// typically out-of-order, other operations that logically come after a call to this method
401    /// could be reordered to come before the measurement, thereby skewing the overall time
402    /// measured.
403    ///
404    /// [`raw`]: Clock::raw
405    pub fn end(&self) -> u64 {
406        match &self.inner {
407            ClockType::Monotonic(monotonic) => monotonic.now(),
408            ClockType::Counter(_, _, counter, _) => counter.end(),
409            ClockType::Mock(mock) => mock.value(),
410        }
411    }
412
413    /// Scales a raw measurement to reference time.
414    ///
415    /// You must scale raw measurements to ensure your result is in nanoseconds.  The raw
416    /// measurement is not guaranteed to be in nanoseconds and may vary.  It is only OK to avoid
417    /// scaling raw measurements if you don't need actual nanoseconds.
418    ///
419    /// Returns an [`Instant`].
420    pub fn scaled(&self, value: u64) -> Instant {
421        let scaled = match &self.inner {
422            ClockType::Counter(_, _, _, calibration) => scale_src_to_ref(value, calibration),
423            _ => value,
424        };
425
426        Instant(scaled)
427    }
428
429    /// Calculates the delta between two measurements, and scales to reference time.
430    ///
431    /// This method is slightly faster when you know you need the delta between two raw
432    /// measurements, or a start/end measurement, than using [`scaled`] for both conversions.
433    ///
434    /// [`scaled`]: Clock::scaled
435    pub fn delta(&self, start: u64, end: u64) -> Duration {
436        // Safety: we want wrapping_sub on the end/start delta calculation so that two measurements
437        // split across a rollover boundary still return the right result.  However, we also know
438        // the TSC could potentially give us different values between cores/sockets, so we're just
439        // doing our due diligence here to make sure we're not about to create some wacky duration.
440        if end <= start {
441            return Duration::new(0, 0);
442        }
443
444        let delta = end.wrapping_sub(start);
445        let scaled = match &self.inner {
446            ClockType::Counter(_, _, _, calibration) => {
447                mul_div_po2_u64(delta, calibration.scale_factor, calibration.scale_shift)
448            }
449            _ => delta,
450        };
451        Duration::from_nanos(scaled)
452    }
453
454    /// Gets the most recent current time, scaled to reference time.
455    ///
456    /// This method provides ultra-low-overhead access to a slightly-delayed version of the current
457    /// time.  Instead of querying the underlying source clock directly, a shared, global value is
458    /// read directly without the need to scale to reference time.
459    ///
460    /// The upkeep thread must be started in order to update the time.  You can read the
461    /// documentation for [`Builder`] for more information on starting the upkeep thread, as well
462    /// as the details of the "current time" mechanism.
463    ///
464    /// If the upkeep thread has not been started, the return value will be `0`.
465    ///
466    /// Returns an [`Instant`].
467    pub fn recent(&self) -> Instant {
468        match &self.inner {
469            ClockType::Mock(mock) => Instant(mock.value()),
470            _ => Instant(GLOBAL_RECENT.load()),
471        }
472    }
473
474    /// Updates the recent current time.
475    ///
476    /// Most callers should use the existing [`Builder`] machinery for spawning a background thread
477    /// to handle upkeep, rather than calling [`upkeep`] directly.
478    pub fn upkeep(value: Instant) {
479        GLOBAL_RECENT.store(value.0);
480    }
481}

489// A manual `Clone` impl is required because `atomic_shim`'s `AtomicU64` is not `Clone`.
490impl Clone for ClockType {
491    fn clone(&self) -> Self {
492        match self {
493            ClockType::Mock(mock) => ClockType::Mock(mock.clone()),
494            ClockType::Monotonic(monotonic) => ClockType::Monotonic(monotonic.clone()),
495            ClockType::Counter(last, monotonic, counter, calibration) => ClockType::Counter(
496                AtomicCell::new(last.load()),
497                monotonic.clone(),
498                counter.clone(),
499                *calibration,
500            ),
501        }
502    }
503}

505/// Sets this clock as the default for the duration of a closure.
506///
507/// This will only affect calls made against [`Instant`].  [`Clock`] is always self-contained.
508pub fn with_clock<T>(clock: &Clock, f: impl FnOnce() -> T) -> T {
509    CLOCK_OVERRIDE.with(|current| {
510        let old = current.replace(Some(clock.clone()));
511        let result = f();
512        let _ = current.replace(old);
513        result
514    })
515}

517/// Sets the global recent time.
518///
519/// While callers should typically prefer to use [`Upkeep`] to establish a background thread in
520/// order to drive the global recent time, this function allows callers to customize how the global
521/// recent time is updated.  For example, programs using an asynchronous runtime may prefer to
522/// schedule a task that does the updating, avoiding an extra thread.
523pub fn set_recent(instant: Instant) {
524    GLOBAL_RECENT.store(instant.as_u64());
525}

527#[inline]
528pub(crate) fn get_now() -> Instant {
529    if let Some(instant) = CLOCK_OVERRIDE.with(|clock| clock.borrow().as_ref().map(|c| c.now())) {
530        instant
531    } else {
532        GLOBAL_CLOCK.get_or_init(Clock::new).now()
533    }
534}
535
536#[inline]
537pub(crate) fn get_recent() -> Instant {
538    // We make a small trade-off here where if the global recent time isn't zero, we use that,
539    // regardless of whether or not there's a thread-specific clock override.  Otherwise, we would
540    // blow our performance budget.
541    //
542    // Given that global recent time shouldn't ever be getting _actually_ updated in tests, this
543    // should be a reasonable trade-off.
544    let recent = GLOBAL_RECENT.load();
545    if recent != 0 {
546        Instant(recent)
547    } else {
548        get_now()
549    }
550}

552#[inline]
553fn scale_src_to_ref(src_raw: u64, cal: &Calibration) -> u64 {
554    let delta = src_raw.saturating_sub(cal.src_time);
555    let scaled = mul_div_po2_u64(delta, cal.scale_factor, cal.scale_shift);
556    scaled + cal.ref_time
557}
558
559#[inline]
560fn mul_div_po2_u64(value: u64, numer: u64, denom: u32) -> u64 {
561    // Modified muldiv routine where the denominator has to be a power of two. `denom` is expected
562    // to be the number of bits to shift, not the actual decimal value.
563    let mut v: u128 = value as u128;
564    v *= numer as u128;
565    v >>= denom;
566    v as u64
567}

569#[allow(dead_code)]
570#[cfg(all(target_arch = "x86_64", target_feature = "sse2"))]
571fn has_tsc_support() -> bool {
572    read_cpuid_nonstop_tsc() && read_cpuid_rdtscp_support()
573}
574
575#[allow(dead_code)]
576#[cfg(not(all(target_arch = "x86_64", target_feature = "sse2")))]
577fn has_tsc_support() -> bool {
578    false
579}
580
581#[cfg(any(target_arch = "x86", target_arch = "x86_64"))]
582fn read_cpuid_nonstop_tsc() -> bool {
583    let cpuid = CpuId::new();
584    cpuid
585        .get_advanced_power_mgmt_info()
586        .map_or(false, |efi| efi.has_invariant_tsc())
587}
588
589#[cfg(any(target_arch = "x86", target_arch = "x86_64"))]
590fn read_cpuid_rdtscp_support() -> bool {
591    let cpuid = CpuId::new();
592    cpuid
593        .get_extended_processor_and_feature_identifiers()
594        .map_or(false, |efi| efi.has_rdtscp())
595}

673    #[test]
674    #[cfg_attr(not(feature = "flaky_tests"), ignore)]
675    #[cfg_attr(
676        all(target_arch = "wasm32", target_os = "unknown"),
677        wasm_bindgen_test::wasm_bindgen_test
678    )]
679    fn test_reference_source_calibration() {
680        let clock = Clock::new();
681        let reference = Monotonic::new();
682
683        let loops = 10000;
684
685        let mut overall = Variance::new();
686        let mut src_samples = [0u64; 1024];
687        let mut ref_samples = [0u64; 1024];
688
689        for _ in 0..loops {
690            for i in 0..1024 {
691                src_samples[i] = clock.now().as_u64();
692                ref_samples[i] = reference.now();
693            }
694
695            let mut last = None;
696            let is_src_monotonic = src_samples.iter().all(|n| match last {
697                None => {
698                    last = Some(n);
699                    true
700                }
701                Some(on) => {
702                    if n >= on {
703                        last = Some(n);
704                        true
705                    } else {
706                        false
707                    }
708                }
709            });
710            assert!(is_src_monotonic);
711
712            let mut last = None;
713            let is_ref_monotonic = ref_samples.iter().all(|n| match last {
714                None => {
715                    last = Some(n);
716                    true
717                }
718                Some(on) => {
719                    if n >= on {
720                        last = Some(n);
721                        true
722                    } else {
723                        false
724                    }
725                }
726            });
727            assert!(is_ref_monotonic);
728
729            let local = src_samples
730                .iter()
731                .zip(ref_samples.iter())
732                .map(|(s, r)| (*s as f64, *r as f64))
733                .map(|(s, r)| (r - s).abs())
734                .collect::<Variance>();
735
736            overall.merge(&local);
737        }
738
739        println!(
740            "reference/source delta: mean={} error={} mean-var={} samples={}",
741            overall.mean(),
742            overall.error(),
743            overall.variance_of_mean(),
744            overall.len(),
745        );
746
747        // If things are out of sync more than 1000ns, something is likely scaled wrong.
748        assert!(overall.mean() < 1000.0);
749    }

751    #[test]
752    #[cfg_attr(not(feature = "flaky_tests"), ignore)]
753    #[cfg_attr(
754        all(target_arch = "wasm32", target_os = "unknown"),
755        wasm_bindgen_test::wasm_bindgen_test
756    )]
757    fn test_reference_self_calibration() {
758        let reference = Monotonic::new();
759
760        let loops = 10000;
761        let samples = 1024;
762
763        let mut overall = Variance::new();
764        let mut deltas = Vec::with_capacity(samples);
765        deltas.reserve(samples);
766
767        for _ in 0..loops {
768            deltas.clear();
769            for _ in 0..samples {
770                let rstart = reference.now();
771                let rend = reference.now();
772
773                deltas.push(rend - rstart);
774            }
775
776            let local = deltas.iter().map(|i| *i as f64).collect::<Variance>();
777            overall.merge(&local);
778        }
779
780        println!(
781            "reference/reference inter-call delta: mean={} error={} mean-var={}",
782            overall.mean(),
783            overall.error(),
784            overall.variance_of_mean()
785        );
786
787        // We should be able to call the reference clock, back-to-back, within 2000 nanoseconds.  If
788        // it's slower than that, something is probably amiss on the system the test is running under.
789        assert!(overall.mean() < 2000.0);
790    }
